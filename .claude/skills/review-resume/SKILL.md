# Resume Review Skill

職務経歴書を読み込み、以下の評価基準で10点満点スコアリングし、markdownレポートファイルとして出力する。

## 入力

`$ARGUMENTS` にファイルパスが渡される。

- ファイルパスが渡されていない場合は「職務経歴書のファイルパスを指定してください」と伝えて終了する。
- PDFの場合は Read ツールで読み込む。テキストファイルの場合も同様。

## 評価項目

以下の6項目をそれぞれ10点満点で評価する。

### 1. Software Engineering の職務経験（3年以上）
- 5名以上の開発チームでの経験があるか
- 新規プロダクト開発・既存プロダクトへの機能追加・既存プロダクトの運用保守、それぞれの経験があるか
- 経験年数が3年以上あるか

### 2. Product Management の職務経験（3年以上）
- 5名以上の開発チームでのPM経験があるか
- 新規プロダクト・機能追加・運用それぞれのフェーズを経験しているか
- 経験年数が3年以上あるか

### 3. Project Management の職務経験（3年以上）
- 開発メンバー10名以上のプロダクトに関わるPjM経験があるか
- 経験年数が3年以上あるか

### 4. インターネット広告業界での職務経験
- 広告配信・DSP/SSP/DMP・アドテク・デジタルマーケティング等に関わる業務経験があるか
- 経験の深さ（表層的な関わりか、コア業務かどうか）

### 5. 日本語でのビジネス会話
- 日本語ネイティブまたはビジネスレベルの記載があるか
- 職務経歴書が日本語で書かれているかも参考にする

### 6. 英語でのビジネス会話
- ビジネスレベルの英語スキルの記載があるか（TOEIC・TOEFL・実務経験等）
- 英語での業務経験（海外チームとの協業、英語でのドキュメント作成等）

## スコアリング基準

| 点数 | 基準 |
|------|------|
| 10 | 条件を完全に満たしており、特筆すべき実績がある |
| 7〜9 | 条件をほぼ満たしているが、一部不足または確認できない情報がある |
| 4〜6 | 条件を部分的に満たしているが、明確な不足がある |
| 1〜3 | 条件をほとんど満たしていないが、関連する経験の断片がある |
| 0 | 記載なし、またはまったく該当しない |

## 出力

以下の形式でmarkdownファイルを作成して保存する。

**ファイル名**: 入力ファイルと同じディレクトリに `review_<元ファイル名（拡張子なし）>.md` として保存する。
例: `/path/to/yamada_taro.pdf` → `/path/to/review_yamada_taro.md`

**内容フォーマット**:

```markdown
# レジュメレビューレポート

- **対象ファイル**: <ファイルパス>
- **レビュー日**: <今日の日付>

---

## 評価サマリー

| 評価項目 | スコア |
|----------|--------|
| Software Engineering 職務経験（3年以上） | X / 10 |
| Product Management 職務経験（3年以上） | X / 10 |
| Project Management 職務経験（3年以上） | X / 10 |
| インターネット広告業界での職務経験 | X / 10 |
| 日本語ビジネス会話 | X / 10 |
| 英語ビジネス会話 | X / 10 |
| **合計** | **XX / 60** |

---

## 詳細評価

### 1. Software Engineering 職務経験（3年以上）：X / 10

**評価根拠**：
- <職歴から読み取れる具体的な記述>

**強み**：
- <プラス要素>

**減点・懸念点**：
- <不足している点や確認できない情報>

---

（以下同様に2〜6を記述）

---

## 総評

<候補者全体の印象、特筆すべき強みと懸念点を3〜5文でまとめる>
```

ファイルを保存したら、保存先パスをユーザーに伝える。
